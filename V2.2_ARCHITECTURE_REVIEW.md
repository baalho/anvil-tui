# V2.2 Architecture Review and Refactor Plan

This document serves as memory and a hand-off guide for Opus (or other agents) regarding the recent fixes and the architectural debt accumulated in `anvil-tui`.

## 1. What We Fixed (The "Kids Mode" Overhaul)

In our recent sessions, we diagnosed and fixed several cascading reliability issues regarding the Sparkle/Bolt kids personas:

1.  **The "Talking but not Doing" Bug**: 
    *   *Issue*: Models (especially smaller ones like Qwen3-Coder) with `tool_choice: auto` would frequently say "Sure, I'll make a game!" but output zero tool calls.
    *   *Fix*: Overrode `Agent::turn` to force `tool_choice: required` if `is_kids_persona` or a `kids-*` skill is active. This forces the model to trigger a tool call on every turn.
2.  **Mode Conflicts**:
    *   *Issue*: `Mode::for_persona` was mapping kids to `Creative` mode, which entirely disabled tool sending.
    *   *Fix*: All personas now default to `Coding` mode, ensuring tools are populated in the LLM context.
3.  **MLX Profile Matching**:
    *   *Issue*: The workspace MLX model profile was being ignored due to a substring mismatch.
    *   *Fix*: Updated `match_patterns` in `qwen3-coder-next.toml` to accurately match `mlx-community/Qwen3...`.
4.  **Kids UI Simplification**:
    *   *Issue*: 7-year-olds were seeing JSON schemas, `exit code: 0`, and `stdout:` prefixes bleeding into their chat.
    *   *Fix*: We added aggressive filtering to the `interactive.rs` loop. Kids now only see "✨ writing some magic code..." instead of the `file_write` schema, and shell metadata is successfully stripped.

## 2. The Devil's Advocate (Shortcuts We Took & Technical Debt)

While the product acts perfectly right now, we compromised the generic, clean agent loop to achieve it. Opus, when you read this, these are the architectural shortcuts you need to fix:

### A. Domain Logic Leaking into `interactive.rs`
**The Debt**: The core terminal event loop (`crates/anvil/src/interactive.rs`) now contains hardcoded strings for unicorns, robots, and magic dust. The generic `TerminalRenderer` shouldn't care who Sparkle is. 
**The Fix**: Refactor `Renderer` to support decorators (e.g., `KidsRenderer` that wraps `TerminalRenderer`), or implement a "System Event Hook" that allows personas to rewrite the `ToolResult` strings *before* it hits the renderer.

### B. The `tool_choice` Hack
**The Debt**: We are dynamically querying `agent.persona().key` and `agent.active_skills()` deep within `Agent::turn` to construct the LLM request. State configuration is fractured across the agent instance. 
**The Fix**: Centralize the configuration state. Bring this into a "Turn Context" object, or allow `.toml` configs (like profiles) to dictate tool enforcement rules, preventing hardcoded Rust logic from deciding when.

### C. Security of the Shell Sandbox
**The Debt**: Kids personas auto-approve every `shell` tool execution to remove UX friction. However, `shell` runs un-sandboxed via `sh -c` natively on the host Mac.
**The Fix**: Either restrict the shell tool to Python execution only (`python3 script.py`), or integrate a lightweight Docker/WASM container for local execution when auto-approve is active.

### D. Sub-Optimal UI Modality
**The Debt**: Relying purely on standard output to view full stack traces, file creations, and artifacts leads to terminal spam. You noted preferring Claude Code CLI or OpenCode. The current UI limits our ability to present complex multi-file diffs cleanly. 
**The Fix**: Fulfill the goal of true Zellij or Neovim integration. For example, instead of dumping output to `stdout`, Anvil should control Zellij panes programmatically—sending the "artifact" to a split pane (Neovim or bat terminal) while keeping the conversational chat loop clean, much like an IDE.
